form태그는 태그안에 input 태그를 두어서 사용자로부터 정보를 입력받아 서버로 보내는 역활을 담당
    form태그가 안맞거나 없으면 서버로 정보를 보낼 수 없음
    에러 - form태그에 정보를 담아 보낼때 실수를 많이 한다.
    보통 하나의 웹페이지에 하나의 form 태그를 두는게 일반적이다.
    여러개 둘 수 있긴 하나, 가급적 하나만 보내자
input태그의name속성=값1&input태그의name속성2=값2&input태그의name속성3=값3 name속성을 키로 하고 value속성에 값이 저장된다 username=홍길동&userage=&useraddress= form태그안에 있어야 서버로 정보를 보낸다. 각 태그를 식별하는 속성값이 3개가 있다. id name class 가 있다. input 태그류만 서버로 정보를 전송할 수 있는 태그들만 name속성을 갖는데, table, div, p, h ... id랑 class만 갖는다
